# Changes — 2026-04-18

Comprehensive rework session. Below is a grouped summary of what shipped.

## Root-cause fixes

### Google Sheets sync was silently broken
- **Problem:** `registrationToSheetRow` wrote 37 fields while `setup-google-sheets.js` only created 26 headers in a different order. Every registration row landed with misaligned columns — email under "Phone", timestamps under "Type", etc.
- **Fix:** rewrote the setup script with a single authoritative 36-column schema, per-column widths, number/date/currency formats, enum dropdowns, checkbox bools, NorTex-gold bold-white centered headers, frozen row, and alternating banding. Removed the dead `Payments` tab.
- **New tooling:** `scripts/verify-sheets.js` performs a read-only schema check; safe to run any time.
- **Dates:** `Created At`, `Updated At`, and `Event Date` now render as human-readable Central-Time strings (`Apr 18, 2026 · 4:53 PM CDT`, `Sat, Jun 20, 2026`) instead of raw ISO. Uses `Intl.DateTimeFormat` with `America/Chicago` tz — auto-switches between CDT/CST.

### Stripe code was dead weight
The live UI uses Square Payment Links, not Stripe. The previous developer's Stripe SDK, webhook code, env vars, and unused `Payments` sheet tab were all orphans.
- **Removed:** `stripe` dependency from `package.json`, `src/lib/sheets/payments.ts`, `src/app/api/payments/confirm/route.ts`'s Stripe-only branches, `PaymentData` type, `stripe_session_id` column (was AK, now the schema is A:AJ · 36 cols), `validateStripeConfig`, and all Stripe env vars.
- **Kept:** the Square success-handler route (`/api/payments/confirm`) — it's alive and wired to the post-payment redirect from Square.
- Also deleted orphan `src/lib/sheets/settings.ts` (zero callers).

### Mobile hamburger menu was broken
- **Bug:** the invisible click-outside overlay (`fixed inset-0 z-40`) was painted on top of the hamburger X button and the mobile Register link, so taps on those elements hit the overlay and closed the menu instead of the intended action.
- **Fix:** lifted the top-bar mobile action cluster with `relative z-50` so it sits above the overlay. Added a visible `bg-black/40` backdrop so the modal state reads clearly. Added `aria-expanded`, `aria-controls`, and `id="mobile-nav"` for accessibility.

## Email system

Before today: customers only got an email *after* paying via Square. No owner notification ever. Template said "please complete payment" even after they had paid.

### New flow
| When | To | Template | Purpose |
|---|---|---|---|
| Form submitted | customer | `buildRegistrationConfirmationEmail` | "Registration received — complete payment" · includes Square link for abandoned-cart recovery |
| Form submitted | admin (`ADMIN_NOTIFICATION_EMAIL`) | `buildOrderNotificationEmail` | Scannable summary for operator — amber "Pending Payment" pill, contact info, roster |
| Post-Square return | customer | `buildPaymentConfirmedEmail` | Green "You're In!" banner, "What's Next" checklist, no payment CTA |

- New env var `ADMIN_NOTIFICATION_EMAIL` mirrored in `.env.local` and Netlify (set to `c.whitworth07@gmail.com`).
- SMTP uses existing Gmail config (`smtp.gmail.com:587` + App Password).
- Email sends are non-blocking — registration succeeds even if SMTP is down. Errors log, don't throw.

## Outstanding follow-ups (flagged, not shipped)

1. **Sheet column order** — currently admin-metadata first. Proposed human-first reorder (Created At · First Name · Last Name · Email · Phone · ...) drafted but not executed — awaiting layout approval.
2. **Apostrophe HTML-entity escaping** — the sanitizer writes `O&#x27;Sullivan` to the sheet instead of `O'Sullivan`. 5-line fix: decode HTML entities before writing, or swap sanitizer.
3. **Team form Notes field** — value not captured when filled. React state binding bug. Individual form Notes field works fine.
4. **Env-gated Square URLs** — currently hardcoded in `PaymentStep.tsx` + email template. Making them env-driven enables safe sandbox testing without editing code.
5. **Payment-confirm security** — `/api/payments/confirm` trusts the `submission_id` alone; a user could skip Square and still mark themselves as paid. Fix via Square webhook on `payment.updated` (requires Square Webhook signing key + public endpoint) or an HMAC nonce between form submit and success page.
6. **Next.js upgrade** — 14.0.0 has security advisories. Dependabot will open PRs.
